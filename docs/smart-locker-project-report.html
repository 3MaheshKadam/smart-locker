1. Abstract / Project Overview

The Smart Locker System is an IoT-based automated locker rental platform that allows users to book, pay for, and unlock physical lockers using a web browser on their mobile phone. The user scans a QR code placed on the locker, which opens a mobile web app. They authenticate via One-Time Password (OTP), select a rental duration, pay online using Razorpay, and the locker unlocks automatically.

The system uses the MQTT protocol to send real-time lock/unlock commands from the web server to an ESP8266 microcontroller that controls a physical solenoid lock. An additional HTTP polling mechanism acts as a backup channel for hardware reliability.

Problem Solved: Traditional lockers require physical keys or tokens. This system eliminates keys entirely — a user's mobile phone and email are sufficient to rent and access a locker from anywhere.

5. MQTT – How Connection is Made (Detail)

5.1 What is MQTT?

MQTT (Message Queuing Telemetry Transport) is a lightweight publish-subscribe messaging protocol designed for IoT devices. It works on a broker-based model:

  • Publisher — sends a message to a topic (web app)
  • Broker — receives and routes messages (HiveMQ)
  • Subscriber — receives messages from a topic (ESP8266)
Analogy: MQTT is like a post office. The web app (publisher) drops a letter in the "locker/L01/cmd" box. HiveMQ (post office/broker) stores and forwards it. ESP8266 (subscriber) checks that box and reads the letter.

5.5 Server-Side MQTT Code Explanation

// lib/mqtt.ts — Server-side MQTT client

const BROKER_URL = 'wss://broker.hivemq.com:8884/mqtt';

// Creates a connection to HiveMQ broker
function getConnectedClient(): Promise<mqtt.MqttClient> {
  return new Promise((resolve, reject) => {
    // Reuse existing connection if already connected
    if (client && client.connected) { resolve(client); return; }

    // Create new client with unique ID
    client = mqtt.connect(BROKER_URL, {
      clientId: `nextjs_server_${Math.random().toString(16).slice(2, 8)}`,
      clean: true,          // No persistent session
      connectTimeout: 10000, // 10 second timeout
      reconnectPeriod: 0,   // No auto-reconnect (serverless environment)
    });

    client.on('connect', () => resolve(client));
    client.on('error', (err) => reject(err));
  });
}

// Publish UNLOCK command to locker
async function publish(locker_id, command) {
  const c = await getConnectedClient();
  const topic = `locker/${locker_id}/cmd`; // e.g. "locker/L01/cmd"

  c.publish(topic, command, { qos: 0, retain: false });
  // QoS 0 = fire and forget (no acknowledgment required)
  // retain: false = don't store message on broker
}

export function publishUnlock(locker_id) { return publish(locker_id, 'UNLOCK'); }
export function publishLock(locker_id)   { return publish(locker_id, 'LOCK'); }

5.6 Browser-Side MQTT Code Explanation

// lib/mqtt-browser.ts — Browser-side singleton MQTT client

// mqtt.js is loaded from CDN in layout.tsx (beforeInteractive)
// So window.mqtt is available in the browser

export function initMqtt() {
  if (client) return; // Already connected — don't create duplicate

  const mqttLib = window.mqtt; // From CDN

  client = mqttLib.connect('wss://broker.hivemq.com:8884/mqtt', {
    connectTimeout: 10000,
    reconnectPeriod: 1000,            // Retry every 1 second if disconnected
    clientId: 'web_' + randomHex(),   // Unique browser client ID
    clean: true,
  });

  // Event listeners update UI state
  client.on('connect',   () => notify('connected'));
  client.on('reconnect', () => notify('reconnecting'));
  client.on('close',     () => notify('disconnected'));
  client.on('error',     () => notify('error'));
}

// Send command from browser to locker
export function sendMqttCommand(lockerId, command) {
  if (!client || !client.connected) return;

  const topic = `locker/${lockerId}/cmd`; // e.g. "locker/L01/cmd"
  client.publish(topic, command, { qos: 0, retain: false });
}

5.7 ESP8266 Arduino Code (Hardware Side)

// ESP8266 Arduino Sketch — Subscribes to MQTT and controls relay

#include <ESP8266WiFi.h>
#include <PubSubClient.h>

// WiFi credentials
const char* ssid     = "YOUR_WIFI_NAME";
const char* password = "YOUR_WIFI_PASSWORD";

// MQTT broker
const char* mqttServer = "broker.hivemq.com";
const int   mqttPort   = 1883; // TCP port (not WebSocket)

// Locker ID (unique per device)
const char* LOCKER_ID = "L01";
const int   RELAY_PIN = D1; // GPIO5

WiFiClient espClient;
PubSubClient mqttClient(espClient);

// ── Callback: called when message arrives on subscribed topic ──
void onMessageReceived(char* topic, byte* payload, unsigned int length) {
  String message = "";
  for (int i = 0; i < length; i++) message += (char)payload[i];

  Serial.println("Message: " + message);

  if (message == "UNLOCK") {
    digitalWrite(RELAY_PIN, LOW);  // LOW = Relay ON = Solenoid energized = UNLOCK
    Serial.println("Locker UNLOCKED");
  }
  else if (message == "LOCK") {
    digitalWrite(RELAY_PIN, HIGH); // HIGH = Relay OFF = Solenoid off = LOCKED
    Serial.println("Locker LOCKED");
  }
}

void setup() {
  Serial.begin(115200);
  pinMode(RELAY_PIN, OUTPUT);
  digitalWrite(RELAY_PIN, HIGH); // Start LOCKED

  // Connect to WiFi
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) { delay(500); }
  Serial.println("WiFi connected");

  // Connect to MQTT broker
  mqttClient.setServer(mqttServer, mqttPort);
  mqttClient.setCallback(onMessageReceived);
}

void loop() {
  // Reconnect MQTT if disconnected
  if (!mqttClient.connected()) {
    String clientId = "esp8266_" + String(LOCKER_ID);
    mqttClient.connect(clientId.c_str());

    // Subscribe to this locker's command topic
    String topic = "locker/" + String(LOCKER_ID) + "/cmd"; // "locker/L01/cmd"
    mqttClient.subscribe(topic.c_str());
  }

  mqttClient.loop(); // Process incoming messages
}

12. Conclusion

The Smart Locker System demonstrates a complete integration of web technology, cloud services, payment gateway, and IoT hardware. Key technical achievements:

  • Passwordless Auth: OTP-based email verification with bcrypt hashing ensures secure identity verification without traditional passwords.
  • Real-time IoT Control: MQTT protocol enables near-instant lock/unlock commands from any browser to physical hardware across the internet.
  • Dual-Channel Reliability: MQTT (fast) + HTTP polling (reliable) ensure the locker command always reaches the ESP8266 hardware.
  • Automated Billing: Razorpay integration with overtime detection handles the complete payment lifecycle including edge cases like grace periods.
  • Serverless Architecture: Next.js API routes on Vercel provide a scalable, cost-effective backend without managing dedicated servers.
  • Mobile-First Design: The entire user journey is optimized for mobile browsers accessed via QR code scan.
Total System Cost (per locker unit): ~₹775 hardware + free-tier cloud services (Vercel, MongoDB Atlas free, HiveMQ public broker) = extremely low operational cost for a fully automated locker system.
